Pi Coding Agent 扩展生态指南 — 从安装到实战

上次分享了怎么给 pi coding agent 配自定义 provider，这次聊聊它的扩展生态——一个被低估但非常强大的特性。

免责声明：本文涉及的 API Key 等敏感信息均已脱敏处理。实际使用请自行替换。

为什么需要扩展

Pi 的设计哲学是”最小核心 + 可插拔”——它本身只有最基本的 read/write/edit/bash 工具，但通过扩展机制，你可以给它加上任意能力：子代理、规划模式、持久记忆、权限保护、LSP 诊断……几乎 Claude Code 有的功能，Pi 都能通过扩展补上。

而且扩展是用 TypeScript 写的，本质上就是一个 npm 包。你甚至可以问 Pi 自己写一个扩展。

@qhn/pi-goal 会在设置目标前先进行一个”访谈”——和用户对齐 outcome、完成标准、决策风格、ask-before 边界——然后再开始自主工作。体验上更接近 Claude Code 的 plan 模式。

注意两个扩展都注册了 /goal 命令，不要同时装。

🔒 权限保护 — 防误操作

这个扩展给 Pi 加了一层权限门控。当模型要执行删除、覆盖等危险操作时，会先请求确认。适合在需要保护的代码库中使用，防止 AI 不小心把不该删的文件删了。

📋 Todo 浮动层

在 Pi 界面中增加一个 todo 列表浮动层，由模型自己维护。多步任务时特别有用——模型可以看到还剩哪些步骤没做，不会跑偏。

📝 任务规划

确定性任务规划 + 规范编排。适合需要严格按步骤执行的场景，Pi 会先生成计划、再逐步执行、最后验证结果。支持将任务分解为可追踪的子任务。

🧠 持久记忆

让 Pi 拥有跨会话记忆能力。之前聊过的上下文、做过的工作会被记住，下次开启新会话时不需要重新交代背景。

小技巧

  1. 扩展不要一次装太多 — 每个扩展都会增加 context 开销。先装最需要的，不够再加
  2. 两个 goal 扩展冲突 — 不要同时装 pi-agent-goal 和 @qhn/pi-goal，它们都注册了 /goal 命令。二选一
  3. models.json 热重载 — 编辑完不需要重启 Pi，/model 或 --model 时会自动重新加载
  4. API Key 别写死在 JSON 里 — 用 $ENV_VAR 引用环境变量，安全且方便切换
  5. 多模型切换 — Pi 支持 Ctrl+L 快速切换模型，可以按任务类型切换（重任务用 Sol，轻量用 Luna）

小结

Pi 的扩展生态远比我想象的成熟。从 goal 模式到子代理，从权限保护到持久记忆，大部分我想要的 Claude Code 功能都能通过扩展补上，而且安装过程极其简单——一行命令 + /reload 搞定。

如果你也在用 AI 编码助手，不妨试试 Pi + 扩展的组合，说不定会有惊喜。
