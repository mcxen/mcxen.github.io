我的 AI 助手 -- Hermes Agent 日常使用记录

从去年开始用 Hermes Agent（Nous Research 出品的 CLI AI 智能体），到现在已经深度嵌入了日常开发和工作流。这篇整理一下它的主要使用场景，也给好奇的人参考。

本文涉及的所有敏感信息（IP、密码、Token、服务器配置等）均已脱敏处理。

什么是 Hermes Agent

Hermes Agent 是一个开源的 CLI AI 智能体，运行在终端里，能直接操控 macOS/Linux 系统——读写文件、执行命令、管理 git、调用 API、搜索网页。它不是”聊天机器人”，而是一个能动手干活的 AI 助手。

它有完整的工具链和技能系统，可以安装各种”技能”来扩展能力。支持多种大模型后端，我主要用 DeepSeek 和 mimo。

场景一：日常开发辅助

代码编写与调试

最常用的场景。Hermes 可以直接操作文件系统，理解项目结构后帮我写代码、修 bug。

我的 Qx 项目（Tauri 桌面应用，现在 v0.6.5）和 Shelfy 项目（文件整理工具），常见工作流：

遇到编译错误，直接把错误日志扔给它，它能逐层追溯、定位问题根因。

代码审查

PR 提交前用 Hermes 做自动审查——检查安全性、代码质量、风格一致性。发现问题后直接 inline comment 到 GitHub PR 上。

AI 编码 CLI 集成

Hermes 不只是自己写代码，还能操作其他 AI 编码工具。我用的 qodercli 就是通过 Hermes 在 tmux 里调度的：

  • 把任务发给 qodercli
  • 监控它的输出
  • 需要时介入修正

这种”AI 操作 AI”的模式对复杂功能开发特别有效。

场景二：DevOps 与服务器管理

我有一台远程 Linux 服务器（frps）和一台飞牛 NAS。Hermes 通过 SSH 直连执行命令：

  • 配置 frp 内网穿透，开机自启
  • 检查服务状态、日志
  • 安装和更新软件包
  • 管理 Memos 自托管实例
  • NAS 文件管理，SMB 挂载排查
  • Cloudflare Pages 部署静态站点

不需要记命令参数，用自然语言描述需求就行。常用的 SSH 模式是 sshpass 加密码直连，避免交互式提示。

场景三：知识管理与笔记

我有自托管的 Memos 实例和 Obsidian 笔记库，Hermes 可以直接读写：

  • 读到有价值的内容，让它记到 Memos
  • 搜索已有笔记做信息关联
  • 维护”记忆系统”——跨会话保持用户偏好、环境配置、项目约定

记忆系统是 Hermes 的特色功能。它会在每次对话时注入之前积累的知识，不用每次重复说”我的博客在哪”、”代理端口是多少”。

场景四：技能系统与自动化

Hermes 有”技能”（Skill）机制，预定义特定任务的工作流程。遇到重复性任务就打包成技能，下次直接复用。

我现在积累的技能包括：

  • 项目发布：版本号同步、编译检查、打 tag、推送
  • 博客维护：写文章、处理图片、生成部署
  • Homebrew Cask 分发：给 Tauri 应用做 brew tap
  • Tauri 发布流程：多文件版本号更新、CI 验证
  • 系统诊断：快速检查磁盘、内存、网络状况
  • 代码审查：安全扫描、质量门禁

把复杂流程封装成技能后，一句话就能启动一整套操作。

场景五：研究与信息获取

调研新技术、查 API 文档、找解决方案时：

  • 搜索网页 + 提取页面内容 + 综合分析
  • 读 arXiv 论文摘要
  • 跟踪技术博客更新（blogwatcher）
  • 跟踪 Polymarket 市场数据

它能把分散的信息合成一个完整的结论，比自己搜索效率高很多。

场景六：创意与内容

  • 用 ComfyUI 生成图片、视频
  • 生成 ASCII art
  • 制作 SVG 信息图和架构图
  • 用 Manim 做数学动画
  • TouchDesigner 视觉创意

场景七：生活工具

  • Apple Reminders – 通过 CLI 管理待办事项
  • iMessage – 发送和接收消息
  • 地图 – 地理编码、POI 搜索、路线规划
  • 邮件 – Himalaya CLI 收发邮件

我的心得

用了一年，最深的体会是：

  1. AI 助手的能力 = 工具链 x 模型智能。光有聪明的模型不够，能直接操作系统才是生产力。
  2. 技能系统的价值被低估了。把重复工作打包成技能，比每次手写 prompt 高效得多。
  3. 记忆系统让 AI 真正”认识”你。它知道你的项目结构、偏好、常用命令，不用每次从零开始。
  4. 不完美，但在进步。长上下文仍有 token 耗尽的问题，复杂代码重构有时需要人工介入。但整体趋势是正向的。

这篇文章本身就是 Hermes 帮我写的，从记忆和会话记录中自动整理成文。
