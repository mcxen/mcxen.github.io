我的 AI 助手 — Hermes Agent 日常使用记录

去年开始接触 Hermes Agent（Nous Research 出品的通用 AI 智能体），到现在已经深度嵌入了我的日常开发和工作流。这篇博客整理一下它的主要使用场景，作为自己的使用记录，也给好奇的人参考。

免责声明：本文涉及的所有敏感信息（IP、密码、Token、服务器配置等）均已脱敏处理。

什么是 Hermes Agent

Hermes Agent 是一个开源的 CLI AI 智能体，运行在终端里，能直接操控 macOS/ Linux 系统——读写文件、执行命令、管理 git、调用 API、搜索网页——它不是”聊天机器人”，而是一个能动手干活的 AI 助手。

它支持多种大模型后端（我主要用 DeepSeek），有工具链和技能系统，可以安装各种”技能”来扩展能力。

场景一：日常开发辅助

代码编写与调试

最常用的场景。Hermes 可以直接操作文件系统，理解项目结构后帮我写代码、修 bug。

比如我正在做的 Qx 项目（一个 Tauri 桌面应用），常见工作流程是：

遇到编译错误，直接把错误日志扔给它，它能逐层追溯、定位问题根因，不像传统搜索引擎那样靠关键词猜。

代码审查

PR 提交前用 Hermes 做自动审查——检查安全性、代码质量、风格一致性。发现问题后直接 inline comment 到 GitHub PR 上，省去人工 review 的重复劳动。

场景二：DevOps 与服务器管理

我需要管理一台远程 Linux 服务器，做 frp 隧道转发、自托管服务等。Hermes 通过 SSH 直连服务器执行命令：

  • 配置 frp 内网穿透
  • 检查服务状态、日志
  • 安装和更新软件包
  • 管理防火墙规则
  • 定时任务维护

不需要记住一长串命令行参数，用自然语言描述需求就行。

场景三：知识管理与笔记

我有个自托管的 Memos 实例（开源 flomo 替代品），Hermes 可以直接读写 memo 笔记。

日常工作流：

  • 读到有价值的内容，让它记到 Memos
  • 搜索已有笔记做信息关联
  • 定期整理归档

它还能维护”记忆系统”——跨会话保持用户偏好、环境配置、项目约定等信息，不用每次重复说。

场景四：技能系统与自动化

Hermes 有”技能”（Skill）机制，可以预定义特定任务的工作流程。遇到重复性任务就打包成技能，下次直接复用。

举个例子，我建了几个日常技能：

  • 发布检查：发布前检查 git 状态、版本号、changelog
  • 博客维护：在 Hexo 博客上写文章、生成、部署
  • 系统诊断：快速检查磁盘、内存、网络状况

把复杂流程封装成技能后，一句话就能启动一整套操作。

场景五：研究与信息获取

当需要调研新技术、查 API 文档、找解决方案时：

  • 搜索网页 + 提取页面内容 + 综合分析
  • 读 arXiv 论文摘要
  • 跟踪技术博客更新

比我自己搜索的效率高很多——它能把分散的信息合成一个完整的结论。

场景六：创意与 Media

偶尔也用 Hermes 做一些创意类工作：

  • 用 ComfyUI 生成图片
  • 生成 ASCII art
  • 制作 SVG 信息图和架构图
  • 操作 TouchDesigner 做视觉创意

我的心得

用了 Hermes 大半年，最深的感觉是：

  1. AI 助手的能力 = 工具链 × 模型智能。光有聪明的模型不够，能直接操作系统才是生产力。
  2. 技能系统的价值被低估了。把重复工作打包成技能，比每次手写 prompt 高效得多。
  3. 不完美，但在进步。长上下文仍有 token 耗尽的问题，复杂代码重构有时需要人工介入。但整体趋势是正向的。

这篇文章本身就是 Hermes 帮我写的，从我的记忆和记录中自动整理成文。🫡
