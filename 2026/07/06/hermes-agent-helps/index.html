Hermes Agent 帮我做了些什么

深度使用 Hermes Agent 已经快一年了，从最初的尝鲜到现在离不开。这篇文章更新一下它帮我搞定的事情，按类别整理。

项目发布一条龙

最频繁的工作就是给 Qx 项目做发布。从 v0.4.36 到现在的 v0.6.5，发布了几十个版本：

  • 自动检测未提交的变更
  • 更新版本号（package.json、Cargo.toml、tauri.conf.json 三处同步）
  • 运行 tsc --noEmit 和 cargo check 验证编译
  • 打 tag、推送到 GitHub
  • 一条命令搞定，遇到网络波动会自动重试

另一个项目 Shelfy（Tauri 文件整理工具）也是同样的流程。后来还搞定了 Homebrew Cask 分发：brew tap mcxen/qx && brew install --cask mcxen/qx/qx。

frp 隧道和远程管理

需要在外面连回家里的 Mac，Hermes 帮我：

  • 检查 frps 服务器（121.40.88.21）是否在线
  • 配置 frpc 客户端，写好配置文件
  • brew services start frpc 开机自启
  • SSH 到远程 Linux 服务器执行各种运维命令

全程不需要手动编辑配置文件，自然语言描述需求就行。

qodercli 自动化

Qx 项目里很多代码改动是通过 qodercli（AI 编码 CLI）完成的，Hermes 负责在 tmux 里操作它：

  • NSPanel 悬浮面板 – macOS 原生无焦点面板实现，这是 Qx 的核心交互
  • 主题切换卡顿修复 – 排查 CSS 变量在暗黑模式下缺失的问题
  • Markdown 渲染升级 – 从 react-markdown + shiki 换成更轻量的方案
  • 流式输出卡死修复 – 重构 Rust 后端的流式处理逻辑
  • 设置面板布局修复 – 导航项固定行高 + label 截断
  • GitHub Actions 步骤进度 – 解析 CI 页面的 HTML 结构，显示真实的步骤完成情况

Memos MCP 服务器

自己搞了个 Memos 实例（121.40.88.21:12009），Hermes 帮我：

  • 搭建 FastMCP 服务器（~/Documents/OpenSpring/memos-mcp/）
  • 直接通过 API 读写 memo 笔记
  • 搜索已有笔记做信息关联

博客维护

Hexo 博客（mcxen.github.io）的日常维护基本都是 Hermes 在跑：

  • 写文章、整理内容
  • HEIC 照片转 JPEG、图片放到 source/images/
  • hexo generate + hexo deploy（走代理 7897）
  • 自动给没有封面的文章选 Unsplash 关键词配封面
  • RSS 全文订阅配置

网络工具和资源下载

  • X/Twitter 视频 – 用 yt-dlp 下载指定推文视频
  • 720yun 全景图 – 下载全景切片并合成为等距柱状投影图
  • NAS 文件管理 – 通过 SMB 挂载飞牛 NAS（10.0.0.124），排查 macOS SMB 连接问题
  • Cloudflare Pages – 部署静态站点，配置自定义域名

DevOps 和 CI

  • Cloudflare Workers – 搭建战术终端风格的 Web 界面
  • Caddy 反代 + ICP 备案 – 踩坑记录，跟阿里云和 Cloudflare 的 DNS 配置斗智斗勇
  • GitHub Actions – 解析 CI 运行详情页的 HTML 结构，实现真实的步骤进度展示

小结

AI 助手已经从聊天玩具变成了真的能干活的生产力工具。从项目发布、代码重构、服务器配置到内容创作，基本上说”帮我搞一下”，它就能自己折腾出来。

最大的变化是：我现在遇到问题的第一反应不是去搜，而是让 Hermes 来处理。

这篇文章本身就是 Hermes 根据记忆和会话记录自动整理成文的。
