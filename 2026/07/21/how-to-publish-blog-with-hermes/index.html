让 Hermes 来做这些事

上面的步骤手动也能做，但用 Hermes 的话，只需要一句话：

“帮我写一篇关于 XXX 的博客发一下”

它会：

  1. 根据记忆和上下文整理内容，写出 Markdown
  2. 自动选合适的封面关键词
  3. 如果有图片需求，帮你转换格式
  4. hexo generate + hexo deploy 一键发布
  5. 部署失败会自动重试

如果你想更新已有的文章，告诉它改什么就行，它会修改后重新部署。

踩过的坑

没代理部署失败：hexo deploy 背后是 git push，不走代理会超时。每次都得带 http_proxy。

HEIC 格式：iPhone 照片直接放博客会显示不出来，必须用 sips 转 JPEG。

source/downloads/ 被渲染：这个目录的文件会被 Hexo 处理成 HTML 页面，不会保留原始格式。需要下载的文件放 source/images/ 或者用 skip_render 配置。

敏感信息泄露：hexo generate 把 Markdown 编译成 HTML，文章里如果有 API key、密码之类的，会直接出现在公开页面上。发布前一定要检查。

封面图不能为空：主题的 post-default-img 配置是空的话，没设 header-img 的文章在首页就是光秃秃的。

一句话总结

用 Hermes 发博客 = 告诉它写什么 + 它搞定剩下的。省心。
