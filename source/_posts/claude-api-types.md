很多人不知道，Claude 的官方 API 其实分为三种来源，它们在价格、可用范围和性能上都有显著区别。

## 第一种：官方 API

从 Anthropic 官网、亚马逊云（AWS Bedrock）或微软云（Azure）出来的 API。

- **价格**：最高，按官方定价走（约 7 人民币 = 1 美元）
- **可用范围**：不受限制，Cursor SDK、OpenClaw、各种 Chatbot 都可以用
- **质量**：最稳定，完整上下文，无额外提示词注入

这是最正规的渠道，适合对质量要求高、预算充足的企业和开发者。

## 第二种：Claude Max 订阅反代

从 Claude Max 订阅反代出来的 API。

- **价格**：中等，比官方便宜
- **可用范围**：**只能用在 Claude Code 里**，其他任何地方都不能用
- **限制**：官方会检测使用场景，发现在非 Claude 应用中使用会封号

中转商也会限制这种 API 的用途，只允许在 Claude Code 中调用。如果你买到了这种 API，拿去用在其他地方，很可能会被封号。

## 第三种：逆向反代 API

从 Cursor、Kiro 等工具里逆向出来的 API。

- **价格**：非常便宜，便宜到不可思议
- **可用范围**：和第一种一样，可以用在任何地方
- **性能**：会差一些

性能差的原因：

1. 逆向后会带一些默认提示词，影响输出质量
2. 第三方应用为了节约成本，可能提供的是不完整上下文

但价格优势巨大——一天消耗一亿 token 都没问题，如果用第一种官方 API，一天得烧几千块钱。

## 总结

| 类型 | 价格 | 可用范围 | 质量 |
|------|------|----------|------|
| 官方 API | 最贵 | 不限 | 最好 |
| Claude Max 反代 | 中等 | 仅 Claude Code | 好 |
| 逆向反代 | 极便宜 | 不限 | 较差 |

这也解释了为什么有些中转站的 API 质量会差一些——关键要看它是这三种里的哪一种。
